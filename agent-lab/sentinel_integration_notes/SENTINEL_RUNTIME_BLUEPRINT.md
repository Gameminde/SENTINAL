# Sentinel Runtime Blueprint

Status: draft. Do not implement advanced runtime capabilities until Agent Lab audits are complete.

## Target Architecture

```text
Sentinel Control
├── Decision Core
│   ├── CueIdea evidence
│   ├── Research agent
│   ├── Multi-agent debate
│   └── GTM decision engine
├── AgentOps Firewall
│   ├── permission policies
│   ├── risk scoring
│   ├── dry-run preview
│   ├── approval gate
│   ├── action trace
│   └── skill scanner
├── Execution Runtime
│   ├── file executor
│   ├── email draft executor
│   ├── browser sandbox executor
│   ├── desktop sidecar
│   └── channel adapters
├── Learning Runtime
│   ├── memory
│   ├── feedback
│   ├── skill proposals
│   ├── prompt improvement
│   └── eval loop
└── Agent Lab Adapters
    ├── OpenClaw-inspired channels
    ├── Hermes-inspired learning
    ├── OpenJarvis-inspired routing
    └── JARVIS-inspired sidecar
```

## Runtime Promotion Rule

Nothing moves from Agent Lab into Sentinel until it has:

- product value;
- capability map;
- failure-mode map;
- Firewall policy;
- dry-run preview design;
- approval rule;
- trace schema;
- eval dataset;
- test result.

## Near-Term Sentinel Priority

Keep Sprint 9 focused on Research Agent quality:

- competitor alternatives;
- pricing references;
- ICP communities;
- prospect source categories;
- objections and buying triggers;
- manual workaround alternatives;
- evidence gaps.

## Later Runtime Candidates

1. Browser sandbox executor.
2. Channel adapters.
3. Skill manifest scanner.
4. Local/cloud model router.
5. Desktop sidecar.

## Keep Disabled

- real email sending;
- browser submit;
- shell execution;
- desktop app control;
- automatic code modification;
- unrestricted sidecar access;
- payment execution.
